cbm_runner.runner

This module is responsible for orchestrating the execution of Carbon Budget Model (CBM) simulations for various scenarios, including baseline and afforestation projects.

Runner

The Runner class orchestrates the execution of Carbon Budget Model (CBM) simulations
class cbm_runner.runner.Runner(config_path, calibration_year, afforest_data, scenario_data, gen_baseline=True, gen_validation=False, sit_path=None)[source]

The Runner class orchestrates the execution of Carbon Budget Model (CBM) simulations for various scenarios, including baseline and afforestation projects. It utilizes annualized afforestation data to give an estimation of carbon stock or flux over a number of specified years (from the calibration year to the target year).

This class leverages various data factories and managers to prepare input data, set up, and execute CBM simulations, ultimately generating outputs such as carbon stocks and fluxes across different scenarios. It manages the creation and organization of simulation input data using specified directory paths and configuration files.

Parameters:

  • config_path (str) – The path to the CBM configuration file.

  • calibration_year (int) – The year used for calibration.

  • afforest_data (AfforestData) – The afforestation data.

  • scenario_data (ScenarioData) – The scenario data.

  • gen_baseline (bool) – A boolean indicating whether to generate baseline data.

  • gen_validation (bool) – A boolean indicating whether to generate validation data.

  • sit_path (str) – The path to the SIT directory.

run_aggregate_scenarios()[source]

Runs aggregate scenarios for forest data.

This method iterates over a set of scenarios and generates carbon stock data for each scenario. It merges the forest data with a baseline forest data, adds selected columns, and drops duplicate columns. The carbon stock data for all scenarios is then concatenated into a single DataFrame.
